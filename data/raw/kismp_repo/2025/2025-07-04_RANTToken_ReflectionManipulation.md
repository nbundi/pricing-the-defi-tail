## 1. Vulnerability Overview

RANT is a reflection token that redistributes a portion of each transfer fee to all holders. The attacker used a PancakeSwap flash loan to borrow a large amount of RANT, temporarily becoming a massive holder, then monopolized the reflection rewards generated by other addresses' transfer transactions before immediately repaying the loan.

## 2. Vulnerable Code Analysis

```solidity
// ❌ Vulnerable logic: reflection rewards are instantly settled proportional to instantaneous balance
contract RANTToken {
    uint256 private _rTotal; // total reflection supply
    mapping(address => uint256) private _rOwned; // reflection balances

    function _transfer(address from, address to, uint256 amount) internal {
        uint256 rAmount = amount * _getRate();
        _rOwned[from] -= rAmount;
        // Fee is reflected to all holders — monopolized when holding large amount via flash loan
        _reflectFee(tFee);
        _rOwned[to] += rAmount - rFee;
    }

    function _reflectFee(uint256 rFee) private {
        _rTotal -= rFee; // total reflection decreases → each holder's proportional share increases
    }
}

// ✅ Fix: flash loan detection or holding-period-based reward calculation
modifier noFlashLoan() {
    require(tx.origin == msg.sender || isWhitelisted[msg.sender], "No flash loan");
    _;
}
```

## 6. Remediation Recommendations

1. **Holding-period-weighted reflection**: Apply a vesting delay on reflection rewards for recently acquired tokens
2. **Flash loan blocking**: Detect and block patterns of large-scale acquire-receive-return within the same transaction
3. **Reflection reward snapshot**: Calculate rewards based on holdings at a block snapshot
4. **Maximum holding ratio cap**: Set an upper limit on the maximum holding proportion for a single address

## 7. Lessons Learned

- The reflection token mechanism is structurally vulnerable when combined with flash loans — because rewards are determined by instantaneous balance.
- When designing tokens, reward distributions based on "instantaneous state" must always account for the flash loan attack vector.
- 311 BNB is a mid-scale loss for a single attack, but the same pattern repeated could collapse an entire token ecosystem.
